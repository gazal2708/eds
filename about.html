About Our BYOM Journey

Transforming Content Delivery

We've successfully implemented Bring Your Own Markup (BYOM) to serve content directly from GitHub, bypassing traditional content management systems.

Our Approach

API-First Configuration

Using the Configuration Service API, we configure content overlays through browser-based API calls, ensuring seamless authentication and rapid deployment.

Version Control

All HTML content lives in GitHub, giving us full version history, pull request workflows, and developer-friendly tooling.

Hybrid Content Model

BYOM overlay allows us to keep Google Drive for certain content while serving custom HTML for specialized pages.

Key Learnings

  • Browser authentication is essential - terminal-based auth tokens don't work for LDAP users
  • Overlay reconfiguration is needed after adding new HTML files, not just code sync
  • Preview API calls can help force immediate content updates
  • No Google Docs required - BYOM works independently when configured as overlay
